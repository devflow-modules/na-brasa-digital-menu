## Scope

### Public customer flow

- public menu (`/na-brasa`)
- product categories and featured products
- product availability (`Product.available`)
- active/inactive products (`Product.active`)
- addons on products
- local cart
- checkout
- order persistence
- WhatsApp redirect (`wa.me`, no Cloud API)

### Store admin (`/admin`, store-scoped)

- database-backed login
- HttpOnly cookie session (JWT)
- role-based permissions
- orders dashboard and order detail
- order status management
- menu management (`/admin/cardapio`)
- category and product management
- active vs available product state
- addon management (`/admin/cardapio/adicionais`)
- Store settings (`/admin/configuracoes`)
- open/closed Store control (`Store.isOpen`)
- delivery/pickup enablement

### Platform admin (`/master`)

- Store overview and metrics
- Store users management (`/master/stores/[storeId]/users`)
- store-scoped user roles (`STORE_OWNER`, `MANAGER`, `OPERATOR`, `KITCHEN`)

### Quality and operations

- Playwright E2E (Chromium)
- GitHub Actions (quality + E2E)
- production smoke checks documented
- MCP usage policy (`docs/development/mcp-usage.md`)
- production checklist (`docs/production-checklist.md`)

## Known limitations

- no WhatsApp Cloud API
- no online payment
- no delivery zones / multiple delivery areas
- no business hours by weekday (free-text `openingHours` only)
- no password reset in the panel
- no Store CRUD in `/master` (users per Store only)
- no file upload / image management in admin
- no order notifications or realtime updates
- no reporting dashboard
- no audit log UI
- `minimumOrderAmountCents` on Store is not editable in `/admin/configuracoes` yet

## Rollback

- promote a previous stable deployment in Vercel if needed
- keep database migrations as-is unless explicitly reviewed (no destructive reset)
- temporarily stop public promotion if a blocker is found
- use `/admin/configuracoes` to close the Store (`isOpen=false`) while fixing issues
